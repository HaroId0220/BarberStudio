## Acuerdos

### Requerimientos identificados y aprobados

1. **Registro y gestión de citas**

   * Crear citas para clientes.
   * Modificar citas existentes.
   * Cancelar citas.
   * Consultar agenda diaria.

2. **Control de anticipos**

   * Registrar anticipos realizados por los clientes.
   * Consultar historial de pagos.
   * Mostrar saldo pendiente.

3. **Avisos a clientes**

   * Enviar recordatorios de citas.
   * Notificar cancelaciones o cambios de horario.
   * Mostrar alertas de próximas citas.

4. **Catálogo de barberos**

   * Registrar barberos.
   * Consultar disponibilidad.
   * Asignar citas a cada barbero.

5. **Reportes básicos**

   * Reporte de citas realizadas.
   * Reporte de ingresos por anticipos.
   * Reporte de servicios realizados por barbero.

### Riesgos identificados

* Posibles cambios de requerimientos durante el desarrollo.
* Dependencia de conexión a internet para acceder al sistema.
* Resistencia inicial de algunos usuarios al uso del sistema digital.
* Disponibilidad limitada para realizar pruebas con usuarios reales.

### Estimación Inicial

* Se estiman 5 historias de usuario principales.
* El proyecto se desarrollará utilizando metodología Scrum.
* El sistema será una aplicación web responsiva enfocada principalmente en dispositivos móviles.

### Validación del Cliente

El dueño del producto revisó los requerimientos descritos y manifestó su conformidad con el alcance funcional presentado. No se identificaron dudas pendientes durante la sesión.
